# 教師行政管理系統 —— Raw Data 格式參考

> 以下展示 3 間不同規模學校的 `CONFIG` 與 `teachersRaw` 設計方式，你可以根據自己學校的實際情況選用或混搭。

---

## 範例一：小型學校（半日制，8 節課，12 位教師）

**學校背景：**
- 小一至小六，每級 2 班（A、B），共 12 班
- 半日制：8:00 - 12:30
- 無早會、無班主任節，純課堂

```javascript
    Wed: ["", "1A", "1A", "P1英備", "", "4C英", "4C英外", "", "", "1B英", "", "", "", "", ""],
    Thu: ["", "1A", "1A", "", "", "4C英", "", "中式/4F", "", "", "4C周", "1A", "", "4C視", ""],

```

**重點：**
- 15 個時段 = 每個陣列 15 個元素
- 包含早會（`ass`）、班主任（`hr`）、午膳（`lun`）、課後（`extra`）
- `busyKeys` 包含所有 5 個班級（A-E）
- 備課用 `"P1英備"`（含「備」字，系統自動視為空堂）
- 小息當值用 `"中式/4F"`（不含 busyKeys，視為職務非課堂）

## 範例三：大型學校（全日制 + 星期六，18 時段，50+ 教師）

**學校背景：**
- 中學，每級 6 班（A-F）
- 星期一至六上課
- 長課日：8:00 - 16:00，共 10 節課

```javascript
        { key: 'p3',  label: '第三節',   time: '10:10-10:55', type: 'lesson' },
        { key: 'p4',  label: '第四節',   time: '10:55-11:40', type: 'lesson' },
        { key: 'lun', label: '午膳',     time: '11:40-12:30', type: 'lunch' },
        { key: 'p5',  label: '第五節',   time: '12:30-13:15', type: 'lesson' },
```

## busyKeys 設計策略

| 時間表填寫方式 | busyKeys 應包含 |
|----------------|-----------------|
| 用「科目名稱」填寫（如中文、英文、數學） | 所有科目簡稱：`["中", "英", "數", "常", "視", "體", "音"]` |
| 用「班級+科目」填寫（如 1A中文、2B數學） | 科目簡稱 + 班級代碼：`["中", "英", "數", "1A", "1B", "2A"...]` |
| 用「班級代碼」填寫（如 1A、2B、3C） | 所有班級代碼：`["1A", "1B", "2A", "2B", "3A"...]` |
| 混合填寫 | 所有可能出現在課堂的關鍵字 |

**特殊規則（無需列入 busyKeys）：**
- `"備"`：任何包含「備」字的內容自動視為空堂
- `""`（空字串）：自動視為空堂

## 常見錯誤與檢查清單

在給我數據之前，請確認：

- [ ] 所有教師的 `Mon` 陣列長度相同
- [ ] 所有教師的 `Mon` 陣列長度等於 `CONFIG.PERIODS` 的數量
- [ ] `Tue`、`Wed`、`Thu`、`Fri` 的長度與 `Mon` 相同
- [ ] `busyKeys` 包含了你時間表中所有表示「有課」的關鍵字
- [ ] 備課內容包含「備」字（這樣系統才會識別為空堂）
- [ ] 沒有使用全形逗號或特殊符號分隔（只用 `"",`）
