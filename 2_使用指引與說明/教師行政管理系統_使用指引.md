# 教師行政管理系統（DemoV3）使用者說明

> 版本：Demo V3（A學校示範版）  
> 適用檔案：`1_範本系統（可直接使用）/教師行政管理系統_demoV3.html`  
> 使用方式：單一 HTML 檔，瀏覽器直接開啟（Chrome / Edge / Safari）

---

## 1. 系統定位

DemoV3 是一個純前端、免安裝、免資料庫的校務排課與代課支援工具，重點功能如下：

- 教師全週時間表查詢（含早會與小息/午膳當值）
- 指定星期與時段即時找出空堂教師
- 當值及職務總覽（課堂模式 + 當值模式）
- 多位教師共同空堂分析（全週）
- 代課推薦與指派單下載（可列印）

---

## 2. 快速開始（3分鐘）

1. 雙擊開啟 `教師行政管理系統_demoV3.html`。  
2. 在上方頁籤切換功能。  
3. 需要代課時，直接去「代課推薦系統」選教師、日期、時段並生成方案。

---

## 3. 六個頁籤點用

### 3.1 學校概覽

- 顯示班級數、教師數、時段架構、主要科目。
- 用於快速確認版本資料是否正確（例如學年、更新日期）。

### 3.2 教師課表

- 選擇教師後，顯示該教師**全週**時間表。
- 表內會同時展示：
  - 課堂內容
  - 早會當值（07:30–07:55）
  - 小息/午膳當值（如有）

### 3.3 查詢空堂老師

- 選擇「星期 + 時段」後按「查詢空堂教師名單」。
- 系統列出該時段完全空堂教師。

### 3.4 當值及職務查詢

- 先選星期，再選模式（早會、小息一/二/三、午膳，或某一節課）。
- 當選「小息一」時，可切換：
  - 晴天
  - 雨天上學期
  - 雨天下學期

### 3.5 共同空堂分析

- 勾選 2 位或以上教師。
- 按「分析共同時段（全週）」後，系統逐日列出共同空堂時段。

### 3.6 代課推薦系統

- 選擇請假教師與日期。
- 勾選要處理的時段（可全選/全清）。
- 生成方案後：
  - 有課或有當值時段會提供候選代課教師
  - 空堂/備課/無當值時段會標示毋需代替
- 完成指派後可按「下載代課指派單」。

---

## 4. DemoV3 顯示規則（重要）

### 4.1 教師代號顯示

- 系統介面統一顯示 `T01`, `T02`... 代號。
- 原始資料可保留老師簡稱，系統會自動轉換成代號顯示。

### 4.2 斜號雙老師支援

- 在當值資料內如出現 `A/B` 或 `A／B`，系統會當作兩位老師處理。
- 影響範圍包括：
  - 當值查詢
  - 教師課表中的當值顯示
  - 代課推薦（誰已當值、誰可代替）
  - 午膳多教師班別顯示

## 5. 資料輸入規格（客製化必讀）

### 5.1 `CONFIG.PERIODS`

- 每個時段包含 `key`, `label`, `time`, `type`。
- `type: 'lesson'` 會計入課務負擔分，影響代課排序。
- 若你調整時段數目（新增/刪除/改 key），`TEACHERS` 內各日資料需跟 `PERIODS.key` 對齊。

### 5.2 `SCHOOL_DATA_VNEXT.TEACHERS`（vNext 格式，建議使用）

每位教師格式：

```javascript
{
  name: "婷",
  Mon: { hr: "1A", p1: "1A人文", br1: "", /* ... */ },
  Tue: { /* periodKey -> content */ },
  Wed: { /* ... */ },
  Thu: { /* ... */ },
  Fri: { /* ... */ }
}
```

規則：

- `""` 代表空堂
- 包含「備」字（例如 `數共備`, `備課`）會視為空堂
- 有課內容需能被 `busyKeys` 命中（如班別、科目字）
- 建議只填 `PERIODS` 內存在嘅 `key`（例如 `hr`, `p1`, `br1`）。系統會自動補齊漏填的時段為空字串。

### 5.3 `DUTY_DATA`

- 小息一格式：`DUTY_DATA['小息一'][天氣模式][星期]`
- 其他當值：`DUTY_DATA[當值名稱][星期]`
- 午膳每項格式：

```javascript
{ cls: "1A", teachers: ["婷", "雅"] }
```

可用以下寫法代表雙老師：

- 陣列：`teachers: ["櫻", "敏"]`
- 斜號字串：`teacher: "櫻/敏"`（系統會自動拆分）

## 6. 代課推薦邏輯（管理層常問）

- 候選人條件：不是請假教師本人，且該時段可代替。
- 排序原則：當日負擔分由低至高。
  - 課堂 = 2 分
  - 當值 = 1 分
- 指派方式：點一下指派，再點同一人可取消。

---

## 7. 常見問題

**Q1：可否放上學校內聯網？**  
A：可以，這是純前端檔案，直接分享 HTML 即可。

**Q2：為何出現「週六/日無需代課」？**  
A：系統預設只處理星期一至五日期。

**Q3：點解某節顯示「無需代課」？**  
A：該節是空堂、備課，或屬無需代替的當值情況。

**Q4：我改完資料後畫面怪怪地？**  
A：先檢查兩點：  
- `SCHOOL_DATA_VNEXT.CONFIG.PERIODS` 的 `key` 是否正確、沒有重複  
- `DUTY_DATA` 老師代稱是否與 `TEACHERS.name` 可對應

---

## 8. 維護建議

- 每次更新資料後，至少測一次：
  - 教師課表（任意 2 位教師）
  - 當值查詢（小息一切換晴/雨）
  - 代課推薦（任意日期）
- 分享給外校前，建議只保留代號顯示，避免暴露真實姓名。

## 9. （開發者）雙檔工作流程：JSON → 打包成單一 HTML

> 目的：你可以用 `school-data.json` 做版本控制/比對/驗證；出貨時仍然只派一個 `.html` 畀學校同事雙擊開啟。

### 9.1 準備資料 JSON

- 參考：`tools/school-data.schema.example.json`
- 建議每間學校用一個 `school-data.json`（以 vNext period-key 格式）

### 9.2 打包（生成可派發單檔）

在 repo 根目錄執行：

```bash
node tools/pack-school-data.mjs --html "1_範本系統（可直接使用）/教師行政管理系統_demoV3.html" --json school-data.json --out "教師行政管理系統_某校_2025-2026.html"
```

生成出嚟嘅 `.html` 就係最終可派發版本（學校端唔需要 `school-data.json`）。
