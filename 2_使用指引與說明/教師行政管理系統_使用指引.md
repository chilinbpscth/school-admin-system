# 教師行政管理系統 —— 使用指引與客製化手冊

> **版本：** 2.0（代號隱私模式）  
> **適用對象：** 小學/中學行政主任、教務組、IT 統籌  
> **技術要求：** 無需伺服器，只需瀏覽器（Chrome / Edge / Safari）

---

## 1. 系統簡介

這是一個**單頁式純前端系統**（Single HTML File），專為學校行政人員設計，用於：

- 快速查閱全校教師課表
- 即時找出指定時段的空堂教師
- 分析特定時段的教師職務分佈
- 尋找多位教師的共同空堂時段
- **智能代課推薦**：自動找出最適合的代課人選，並生成可列印的指派單

**最大優點：** 無需安裝、無需資料庫、無需網絡（開啟後可離線使用），資料直接寫在檔案內，私隱度極高。

## 2. 快速上手（5分鐘開始）

### 第一步：取得檔案
你會得到一個 `.html` 檔案，例如：
```
teacher_admin_system.html
```

### 第二步：開啟系統
直接用瀏覽器**雙擊開啟**該檔案即可。不需要架設網站、不需要後端伺服器。

### 第三步：瀏覽功能
頂部導航列有 6 個功能頁籤：

| 頁籤 | 用途 |
|------|------|
| **學校概覽** | 顯示班級數、教師人數、時段架構、科目列表 |
| **教師課表** | 查詢任何教師在任何星期的全日時間表 |
| **查詢空堂老師** | 選擇「星期 + 時段」，即時列出無課的教師 |
| **查詢該時段教師職務** | 查看某時段所有教師正在做什麼（如：帶課、小息當值、樓層巡視） |
| **共同空堂分析** | 勾選多位教師，找出大家都有空的時段（方便開會） |
| **代課推薦系統** | 輸入請假教師 + 日期 + 節次，系統自動推薦代課人選並生成簽收單 |

## 3. 功能詳解

### 3.1 學校概覽
- 顯示基本統計數據（班級數、教師人數、時段數）
- 列出每日小息時間安排
- 以標籤形式展示全校開設科目
- **如何修改：** 見下文「客製化教學」第 4.1 節

### 3.2 教師課表查詢
- 從下拉選單選擇教師（顯示為代號或簡稱）
- 選擇星期（一至五）
- 表格即時顯示該教師當天所有時段的內容
- 有課的時段以藍色標示，空堂以灰色顯示

### 3.3 查詢空堂教師
- 選擇「星期」和「時段」
- 點擊「查詢代號名單」
- 系統列出所有在該時段「無課」或「備課」的教師
- 每張卡片顯示教師代號及其當時狀態（如「備課」或「完全空堂」）

### 3.4 查詢該時段教師職務
- 選擇「星期」和「時段」
- 點擊「分析職務分佈」
- 系統按職務類型分組顯示（例如：「1A中文」有 T01、T02；「P1-P2樓層」有 T03）
- 適合檢查小息當值、樓層巡視等安排

### 3.5 共同空堂分析
- 在教師列表中勾選至少 2 位教師（按住思維可複選）
- 選擇星期
- 點擊「分析共同時段」
- 系統找出該天所有「被選教師同時空堂」的時段
- **應用場景：** 安排級會、備課會、家長面談時段

### 3.6 代課推薦系統（核心功能）

#### 操作步驟：
1. **選擇請假教師代號**
2. **選擇請假日期**（系統自動計算是星期幾，週六日會自動拒絕）
3. **選擇代課節次範圍**
   - 可點擊「全選」或「全清」
   - 預設只勾選「課堂類型」時段（非小息、非午膳）
4. **點擊「生成代課方案」**

#### 系統如何推薦代課人選：
- **篩選條件：** 排除請假教師本人；該時段必須空堂
- **排序邏輯：** 按「當天課堂負荷量」由低至高排序（課最少的教師優先顯示）
- **指派方式：** 點擊教師卡片即可指派；再次點擊取消
- **智能判斷：** 若該時段請假教師本身無課（空堂/備課），系統會標示「無需代課」

#### 下載代課指派單：
- 指派完成後，點擊底部「下載代課指派單」按鈕
- 生成 `.html` 檔案，內含：
  - 請假教師代號、日期、星期
  - 每個節次的時間、原任內容、代課教師代號
  - **簽署確認欄**（供代課教師簽名）
  - 內建「列印此通知單」按鈕
- 檔名格式：`代課指派單_T01_2026-05-04.html`

## 4. 數據客製化教學（給不同學校使用）

要將此系統套用至你的學校，只需要修改 HTML 檔案內的 **JavaScript 數據區**。你不需要懂編程，只需按照格式替換文字即可。

### 4.1 修改學校基本資料（學校概覽頁）

在 HTML 內找到 `<section id="section-overview">`，修改以下內容：

```html
<!-- 統計數字 -->
<div class="text-5xl font-bold text-blue-900 mt-3">30 班</div>   <!-- 改為你的班級數 -->
<div class="text-5xl font-bold text-emerald-900 mt-3">44 位</div> <!-- 改為你的教師人數 -->
<div class="text-5xl font-bold text-amber-900 mt-3">16 時段</div> <!-- 改為你的時段數 -->

<!-- 小息時間 -->
<div class="text-sm text-orange-500">10:20 – 10:40</div>  <!-- 改為你的小息時間 -->

<!-- 科目標籤 -->
<span class="px-4 py-1.5 ...">中文 / 中作</span>  <!-- 增刪科目名稱 -->
```

### 4.2 修改時段架構（最重要）

在 `<script>` 開頭找到 `CONFIG.PERIODS`，這是定義學校一日時間表的陣列：

```javascript
const CONFIG = {
    PERIODS: [
        { key: 'pre',  label: '課前',     time: '07:40–08:00', type: 'duty' },
        { key: 'ass',  label: '早會',     time: '08:00–08:15', type: 'ass' },
        { key: 'hr',   label: '班主任節', time: '08:15–08:25', type: 'hr' },
        { key: 'p1',   label: '第一節',   time: '08:25–09:05', type: 'lesson' },
        { key: 'p2',   label: '第二節',   time: '09:05–09:45', type: 'lesson' },
        // ... 繼續添加你的時段
    ],
    DAYS: ["星期一", "星期二", "星期三", "星期四", "星期五"],
    busyKeys: ["中", "英", "數", "常", "普", "視", "體", ...]
};
```

**欄位說明：**

| 欄位 | 說明 | 可選值 |
|------|------|--------|
| `key` | 唯一識別碼（英文，不可重複） | `'p1'`, `'break1'`, `'lunch'` 等 |
| `label` | 顯示名稱 | `'第一節'`, `'小息'`, `'午膳'` 等 |
| `time` | 時間範圍 | `'08:25–09:05'` 或留空 `''` |
| `type` | 時段類型 | `lesson`（課堂）、`duty`（職務）、`ass`（集會）、`hr`（班主任）、`lunch`（午膳）、`extra`（課後） |

**注意：** `type: 'lesson'` 的時段會被計入「課堂負荷量」，影響代課推薦的排序。非課堂時段（小息、午膳）不應設為 `lesson`。

### 4.3 修改 `busyKeys`（忙碌關鍵字）

`busyKeys` 是系統判斷「這個時段有沒有課」的關鍵字列表。

**運作原理：** 若教師某時段的內容**包含**任何一個 `busyKeys` 的字，系統就認為該教師在忙碌（有課）。

**例子：**
```javascript
busyKeys: ["中", "英", "數", "常", "普", "視", "體", "1A", "1B", "2A"]
```
- 若時間表內容是 `"1A中文"` → 包含 `"中"` 和 `"1A"` → **有課**
- 若時間表內容是 `"(P1-P2樓層)"` → 不包含任何 busyKeys → **視為職務/空堂**
- 若時間表內容是 `""`（空）或 `"P1中備"`（包含「備」字）→ **空堂**

**設定建議：**
- 如果你的時間表用**科目名稱**填寫（如中文、英文、數學），`busyKeys` 應列出所有科目簡稱
- 如果你的時間表用**班級代碼**填寫（如 1A、2B、3C），`busyKeys` 應列出所有班級代碼
- **「備」字是特殊關鍵字**：任何包含「備」的內容（如「P1中備」、「備課」）會**強制視為空堂**，無需列入 `busyKeys`

### 4.4 輸入教師課表數據（核心）

在 `teachersRaw` 陣列中，每個教師是一個物件：

```javascript
const teachersRaw = [
    {
        name: "T01",  // 教師顯示名稱（可用代號或真名）
        Mon: ["靜觀行/早課", "1A", "1A", "1A中", "", "", "1A中", "P5中備", "", "5B中", "5B中", "5B", "(P1-P2樓層)", "", "5B中功", ""],
        Tue: ["Mindfulness Walking", "1A", "1A", "5B中", "", "", "P1中備", "1A中", "", "5B中", "2B普", "", "(P3-P6操場)", "", "", ""],
        // ... Wed, Thu, Fri
    },
    // 下一個教師
];
```

**格式規則：**

1. **陣列長度必須與 `CONFIG.PERIODS` 完全一致**
   - 若有 16 個時段，則 `Mon`、`Tue`... 每個陣列都必須有 **16 個元素**
   - 索引 0 對應第一個時段，索引 1 對應第二個時段，如此類推

2. **內容填寫方式：**
   - `""`（空字串）= 完全空堂
   - `"中文"`、`"1A"`、`"數學"` = 有課（會匹配 `busyKeys`）
   - `"P1中備"`、`"備課"` = 備課（視為空堂，無需代課）
   - `"(P1-P2樓層)"`、`"操場"` = 職務/當值（不會匹配 busyKeys，視為忙碌但非課堂）

3. **教師名稱模式：**
   - **隱私模式（推薦）：** `name: "T01"`、`"T02"`... 分享時不洩露真實姓名
   - **簡稱模式：** `name: "陳"`、`"梁"`、`"何"`... 只顯示姓氏
   - **全名模式：** `name: "陳大文"`... 不建議用於分享檔案

### 4.5 修改頁籤名稱與圖標

找到 `TAB_CONFIG`：

```javascript
const TAB_CONFIG = [
    { id: 'section-overview', label: '學校概覽', icon: 'fa-school' },
    { id: 'section-timetable', label: '教師課表', icon: 'fa-clock-rotate-left' },
    // ...
];
```
- `label`：頁籤顯示文字
- `icon`：FontAwesome 圖標名稱（可到 fontawesome.com 搜尋）

### 4.6 修改頁首資訊

找到 `<header>` 區域：
```html
<h1>2025 – 2026年度 教師行政管理系統</h1>
<p>系統開發：張定恒主任 (佛教志蓮小學)</p>
<span>數據版本：2026-05-04 版 (已更新學校資料)</span>
<span>全日 16 時段架構 (44位代號教師)</span>
```
請按你的學校年度、開發者名稱、學校名稱、數據更新日期進行修改。

## 5. 進階設定

### 5.1 更改系統顏色主題

系統使用 Tailwind CSS 顏色類別：
- **藍色系：** `blue-600`、`blue-900`、`indigo-900`（主要介面）
- **綠色系：** `green-600`、`green-800`（空堂查詢）
- **紅色系：** `red-600`、`red-800`（代課系統）
- **青色：** `teal-600`、`teal-800`（職務查詢）

你可以將這些顏色名稱替換為其他 Tailwind 顏色（如 `purple`、`rose`、`cyan`）來改變主題。

### 5.2 調整代課推薦排序邏輯

在 `renderCandidates()` 方法中，預設按「當天課堂數量」升冪排序（課少的優先）：

```javascript
.sort((a, b) => a.load - b.load)
```

你可以改為其他邏輯，例如：
- 按代號順序：`.sort((a, b) => a.name.localeCompare(b.name))`
- 隨機排序：`.sort(() => Math.random() - 0.5)`

### 5.3 列印樣式

系統已內建 `@media print` 規則：
- 列印時會自動隱藏導航列、按鈕、頁首頁尾（帶有 `no-print` 類別的元素）
- 代課指派單本身已優化為 A4 列印格式

## 6. 常見問題（FAQ）

**Q1: 我可以將這個檔案放在學校內聯網分享給其他老師用嗎？**  
A: 可以。將 `.html` 檔案放在學校共用資料夾或內聯網伺服器，其他老師用瀏覽器開啟即可。由於是純前端，不會對伺服器造成負擔。

**Q2: 資料會不會外洩？**  
A: 不會。所有數據都在該 `.html` 檔案內，沒有連接到任何外部資料庫或雲端。建議使用「代號模式」（T01、T02）而非真實姓名，以保護私隱。

**Q3: 我想加一位新老師，怎麼做？**  
A: 在 `teachersRaw` 陣列末尾新增一個物件，確保 `Mon` 到 `Fri` 每個陣列都有與 `CONFIG.PERIODS` 相同數量的元素。

**Q4: 系統顯示「週六/日無需代課」是什麼意思？**  
A: 代課系統只接受星期一至五的日期。如果你選擇了週六或週日，系統會拒絕並提示此訊息。

**Q5: 為什麼某些時段顯示「無需代課」？**  
A: 系統檢查到該時段請假教師本身是空堂或備課（內容為 `""` 或包含「備」字），因此無需安排代課。

**Q6: 我學校有星期六上課，可以修改嗎？**  
A: 可以。找到 `CONFIG.DAYS`，加入 `"星期六"`，然後在每位教師的數據中加入 `Sat: [...]` 陣列。同時需要修改 `getSchedule()` 方法中的星期對應表。

**Q7: 可以將這個系統放在手機上使用嗎？**  
A: 可以。介面已使用響應式設計（RWD），在手機上會自動調整為單欄佈局。

## 7. 技術規格

| 項目 | 規格 |
|------|------|
| 檔案格式 | `.html`（單一檔案） |
| 檔案大小 | 約 50–100 KB（視教師數量而定） |
| 瀏覽器要求 | Chrome 90+ / Edge 90+ / Safari 14+ / Firefox 88+ |
| 網絡要求 | 初次載入需網絡（載入 Tailwind 與 FontAwesome CDN）；開啟後可離線使用 |
| 離線使用 | 可。若需完全離線，可下載 Tailwind CSS 與 FontAwesome 到本機並修改 `<script>` 與 `<link>` 路徑 |
| 數據儲存 | 無。所有資料寫死在 HTML 內，重新整理頁面不會遺失（因為沒有暫存需要保存） |
| 代課指派記錄 | 無法保存。指派結果僅在當前瀏覽 session 有效，下載指派單後即完成記錄 |
