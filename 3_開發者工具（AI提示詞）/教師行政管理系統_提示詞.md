# 教師行政管理系統 —— AI 生成提示詞（Meta Prompt）

> **用途：** 將以下提示詞餵給任何大型語言模型（Claude、GPT、Gemini 等），即可生成一個功能完整的單頁式教師行政管理系統 HTML 檔案。  
> **適用對象：** 學校 IT 統籌、行政主任、或任何想為自己學校客製化此系統的用戶。

---

## 提示詞正文

```markdown
請為我創建一個「教師行政管理系統」，這是一個單頁式純前端 HTML 應用程式（Single-File Application）。
所有 HTML、CSS、JavaScript 都必須封裝在同一個 .html 檔案內，無需後端伺服器，用瀏覽器雙擊開啟即可運行。

## 技術要求

1. **樣式框架：** 使用 Tailwind CSS CDN (`https://cdn.tailwindcss.com`)
2. **圖標：** 使用 FontAwesome 6 (`https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css`)
3. **字體：** 使用 Noto Sans TC (Google Fonts)
4. **語言：** 繁體中文介面 (zh-HK)
5. **架構：** 使用原生 JavaScript ES6 Class，分為 DataManager（數據層）與 App（應用層）
6. **響應式：** 必須支援手機與桌面版（RWD），導航列自動換行

## 數據結構規格

### CONFIG 配置物件
```javascript
const CONFIG = {
    PERIODS: [
        // 每個時段必須包含：key（唯一識別碼）, label（顯示名稱）, time（時間範圍）, type（類型）
        // type 可選值：'lesson'（課堂）、'duty'（職務/當值）、'ass'（集會/早會）、'hr'（班主任節）、'lunch'（午膳）、'extra'（課後）
        { key: 'pre', label: '課前', time: '07:40–08:00', type: 'duty' },
        { key: 'ass', label: '早會', time: '08:00–08:15', type: 'ass' },
        { key: 'hr', label: '班主任節', time: '08:15–08:25', type: 'hr' },
        { key: 'p1', label: '第一節', time: '08:25–09:05', type: 'lesson' },
        // ... 以此類推，定義全日所有時段
    ],
    DAYS: ["星期一", "星期二", "星期三", "星期四", "星期五"],
    busyKeys: ["中", "英", "數", "常", "普", "視", "體", "音", "1A", "1B", "2A", "2B", "3A", "3B", "4A", "4B", "5A", "5B", "6A", "6B"]
    // busyKeys 是用來判斷「有課」的關鍵字。若時間表內容包含任何 busyKeys，即視為忙碌。
    // 特殊規則：內容包含「備」字（如 "P1中備"、"備課"）一律視為空堂，無需列入 busyKeys。
};
```

## 必備功能（6個頁籤）

### 1. 學校概覽頁 (section-overview)
- 頂部顯示 3 個統計卡片：班級數目、教師人數、時段架構
- 中間顯示每日小息安排（時間 + 圖標）
- 底部以標籤形式列出主要科目 / 學習領域
- 頁尾顯示數據來源、更新日期、私隱聲明

### 2. 教師課表查詢 (section-timetable)
- 兩個下拉選單：選擇教師、選擇星期
- 即時渲染表格，欄位包括：時段、時間、內容、類別
- 有課的時段用深色字體，空堂用淺灰色並顯示 "---"

### 3. 查詢空堂教師 (section-free)
- 兩個下拉選單：選擇星期、選擇時段
- 點擊「查詢代號名單」按鈕
- 結果以卡片形式顯示，每張卡片顯示：教師代號、該時段的實際內容（如「備課」或「完全空堂」）
- 無結果時顯示「此時段無空堂教師」

### 4. 查詢該時段教師職務 (section-duties)
- 兩個下拉選單：選擇星期、選擇時段
- 點擊「分析職務分佈」按鈕
- 將該時段所有教師的非空內容按職務類型分組顯示
- 例如：「1A中文」有 T01、T02；「(P1-P2樓層)」有 T03
- 無結果時顯示「此時段無具體職務紀錄」

### 5. 共同空堂分析 (section-common)
- 上方顯示所有教師的勾選框（grid 佈局，可捲動）
- 下拉選單選擇星期
- 點擊「分析共同時段」按鈕
- 若勾選少於 2 人，顯示錯誤提示「請至少選擇兩位代號教師」
- 找出所有被選教師同時空堂的時段，以卡片形式列出時段名稱與時間
- 無共同空堂時顯示「所選代號無共同空堂」

### 6. 代課推薦系統 (section-sub) —— 核心功能
- **輸入區：**
  - 下拉選單：請假教師代號
  - 日期選擇器：請假日期（預設今天）
  - 節次選擇：以勾選框顯示所有時段，預設只勾選 type === 'lesson' 的時段。提供「全選」與「全清」按鈕。
  - 紅色大按鈕：「生成代課方案」
- **邏輯：**
  - 根據日期自動計算星期幾。若為週六或週日，顯示錯誤提示「週六/日無需代課」並終止。
  - 對每個勾選的節次：
    - 若請假教師該時段本身為空堂或備課（isLessonFree === true），顯示「空堂 / 備課 (無需代課)」並置灰
    - 若有課，顯示原課內容，並列出候選代課教師
  - **候選人排序邏輯：**
    - 篩選條件：不是自己；該時段 isLessonFree === true
    - 排序：按當天課堂負荷量（workload）由低至高。workload 只統計 type === 'lesson' 且非空堂的時段數量。
    - 顯示：代號 + 今日課量（如「今日課量: 3節」）
  - **指派機制：**
    - 點擊候選人卡片即可指派（卡片變藍色）。再次點擊同一卡片取消指派。
    - 指派狀態顯示「已指派: T05」或「未指派」
    - 指派記錄保存在 App.subAssignments 物件中（key 為節次 key, value 為教師代號）
- **下載指派單：**
  - 底部提供「下載代課指派單 (代號版)」按鈕
  - 生成一個獨立的 HTML 字串，內含：
    - 標題：代課指派通知單 (簽收用)
    - 請假教師代號、日期、星期
    - 表格：節次、時間時段、原任內容、代課教師(代號)、簽署確認欄
    - 頁尾：教務主任簽名欄、日期
    - 內建「列印此通知單」按鈕
  - 使用 Blob + URL.createObjectURL 觸發下載，檔名：`代課指派單_{代號}_{日期}.html`

## UI / UX 要求

1. **頁首 (Header)：**
   - 深藍色漸層背景 (`bg-gradient-to-r from-blue-900 to-indigo-900`)
   - 左側：系統名稱 + 開發者資訊（使用 badge 樣式）
   - 右側：數據版本標籤 + 系統規格標籤（如「全日 16 時段架構 (44位代號教師)」）
   - 固定在頂部 (sticky, z-50)

2. **導航列 (Nav)：**
   - 白色背景，底部邊框
   - 頁籤使用按鈕形式，帶有 FontAwesome 圖標
   - 當前頁籤高亮：`tab-active` 類別（淺藍底 + 藍色底部邊框）
   - 自動換行 (flex-wrap)
   - sticky 定位，跟在 header 下方

3. **內容區 (Main)：**
   - 每個 section 預設隱藏 (`hidden`)，透過 switchTab 切換
   - 每個 section 使用圓角卡片 (`rounded-2xl`)，帶有對應顏色的邊框
     - 學校概覽：slate-200
     - 教師課表：slate-200
     - 空堂查詢：green-100
     - 職務查詢：teal-100
     - 共同空堂：indigo-100
     - 代課系統：red-100

4. **頁尾 (Footer)：**
   - 深色背景，顯示版權與私隱聲明

5. **Toast 通知：**
   - 固定在右上角 (fixed top-6 right-6)
   - 成功訊息為綠底，錯誤訊息為紅底
   - 帶有滑入滑出動畫 (CSS transition)

6. **列印優化：**
   - 所有帶有 `no-print` 類別的元素在列印時隱藏
   - 代課指派單本身為純黑白表格，適合 A4 列印

## 核心演算法規格

### DataManager 類別必須實現以下方法：

1. **constructor()**
   - 將 teachersRaw 轉換為內部 teachers 陣列
   - 確保 name 欄位被正確覆蓋（支援代號模式）
   - 加入 id 與 displayName

2. **getSchedule(name, day)**
   - 將中文星期轉換為英文鍵名：`{"星期一":"Mon", "星期二":"Tue", ...}`
   - 返回該教師當天的時段陣列，若找不到教師返回 null

3. **isLessonFree(name, day, pIdx)**
   - 取得指定教師、指定星期、指定時段索引的內容
   - 若內容為空字串 `""` 或包含 `"備"` 字 → 返回 true（空堂）
   - 若內容包含任何 CONFIG.busyKeys 中的關鍵字 → 返回 false（有課）
   - 否則 → 返回 true（視為職務或空堂）

4. **getWorkload(name, day)**
   - 遍歷當天所有時段
   - 只統計 `CONFIG.PERIODS[idx].type === 'lesson'` 的時段
   - 若 `isLessonFree(name, day, idx) === false` 則計數 +1
   - 返回總課堂數（整數）

### App 類別必須實現以下方法：

- `init()`：初始化所有下拉選單、頁籤、事件監聽，預設顯示「學校概覽」
- `switchTab(id)`：切換 section 顯示與頁籤高亮
- `populateSelect(id, items, values)`：填充下拉選單
- `showTeacherTimetable()`：渲染教師課表表格
- `findFreeTeachers()`：查詢並渲染空堂教師卡片
- `findDuties()`：分組渲染職務分佈
- `findCommonFree()`：分析並渲染共同空堂
- `generateSubRecommendations()`：生成代課方案區塊
- `renderCandidates(day, pIdx, pkey)`：渲染候選代課教師並按 workload 排序
- `toggleAssign(pkey, name)`：切換指派/取消指派，並重新渲染
- `toggleAllSubPeriods(checked)`：全選/全清代課節次
- `downloadReceipt()`：生成並下載代課指派單 HTML
- `showMessage(message, type)`：顯示右上角 Toast（type: 'success' | 'error'）

## 輸出格式

請直接輸出完整的 HTML 程式碼，格式如下：
- `<!DOCTYPE html>`
- `<html lang="zh-HK">`
- 所有 CSS 放在 `<style>` 標籤內
- 所有 JavaScript 放在 `<script>` 標籤內
- 使用註解標示各區域（如 `<!-- Header -->`、`<!-- Substitution System -->`）
- 在 `teachersRaw` 中填入合理的示範數據（約 5-10 位教師即可），讓我可以直接測試
- 確保 `CONFIG.PERIODS` 定義至少 8 個時段，包含課堂與非課堂類型

## 額外要求

1. 不要將教師真實姓名寫死，使用代號模式（T01、T02）作為預設
2. 確保 `CONFIG.busyKeys` 包含常見科目簡稱與班級代碼
3. 所有按鈕必須有 hover 效果 (`hover:xxx`)
4. 所有卡片必須有輕微陰影 (`shadow-sm` 或 `shadow-md`)
5. 確保在 375px 寬度（手機）下仍可正常操作，不會出現嚴重排版錯誤

## 使用這個提示詞的建議

### 步驟 1：餵給 AI
將上方提示詞（不含最外層的 ```markdown 標記）貼到 Claude、ChatGPT、Gemini 等 AI 對話框中。

### 步驟 2：獲取 HTML
AI 會生成一段完整的 HTML 程式碼。請確保它包含：
- 完整的 `<!DOCTYPE html>` 開頭
- `<script src="https://cdn.tailwindcss.com"></script>`
- 一個 `teachersRaw` 陣列（內有示範數據）
- `DataManager` 與 `App` 兩個 class

### 步驟 3：保存並測試
1. 將程式碼複製到記事本
2. 另存新檔為 `teacher_system.html`
3. 用瀏覽器雙擊開啟
4. 測試每個頁籤功能是否正常

### 步驟 4：客製化
參考《教師行政管理系統_使用指引.md》中的「數據客製化教學」，將 `teachersRaw` 與 `CONFIG` 替換為你學校的實際資料。
